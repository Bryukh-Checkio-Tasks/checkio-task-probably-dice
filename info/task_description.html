For this task, you're on your way to a board game convention and want to learn more about dice probabilities since you suspect you'll be rolling a lot of them soon.

The most common way to use multiple dice is to simply roll them all and add up all the results. To get started with your investigation of dice, write a function that takes the number of dice, the number of sides per die and a target number and returns the probability (to 4 decimal places) of getting a total roll of exactly the target value. For example, if you roll 2 six-sided dice, the probability of getting exactly a 3 is 2/36 or 5.56%, which you should return as 0.0556.

For each test, assume all the dice are the same and are numbered from 1 to the number of sides, inclusive. So a 4-sided die would have an equal chance of rolling a 1, 2, 3 or 4. A 20-sided die would have an equal chance of rolling any number from 1 to 20.

Input: Three integers: the number of dice N, the number of sides per die S and the target number T.

Output: The probability of getting exactly T on a single roll of N S-sided dice, to four decimal places.

Example: 

checkio(2, 6, 3) == 0.0556  # 2 six-sided dice have a 5.56% chance of totalling 3
checkio(2, 6, 4) == 0.0833 
checkio(2, 6, 7) == 0.1667
checkio(2, 3, 5) == 0.2222  # 2 three-sided dice have a 22.22% chance of totalling 5
checkio(2, 3, 7) == 0       # The maximum you can roll on 2 three-sided dice is 6
checkio(3, 6, 7) == 0.0694
checkio(10, 10, 50) == 0.0375

How it is used: This task illustrates some of the basics of probability. Many events can be described as the combination of other events. In this case you're combining several dice into one total.

Preconditions:
1 ≤ N ≤ 10
2 ≤ S ≤ 20
